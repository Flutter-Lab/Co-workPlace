Feature Integration Prompt - Goal Tracker for Existing App
============================================================

Reference:
- Google Sheet: https://docs.google.com/spreadsheets/d/1q9HLfkYi_zSwiXDXJk7Q_vJbkQ-UiMSM5win88WteDg/edit?usp=sharing
- Please review the `Books` tab first and use it as the main inspiration.

Prompt:
You are an expert Flutter developer.

I do NOT want a new standalone app right now.
I want you to add a new `Goal Tracker` feature inside one of my existing Flutter apps.
This feature should be inspired by the `Books` tracking tab in the Google Sheet above.

Main Objective:
Add a reusable goal-tracking module that feels like the book reading tracker, but works inside my current app with minimal disruption to the existing codebase.

Important Rules:
1. Do not create a separate new app.
2. Reuse the existing app architecture, theme, navigation, folder structure, and state management where possible.
3. Keep code changes minimal and focused.
4. Do not do a large refactor unless absolutely necessary.
5. Match the existing app's UI style so the new feature feels native to the app.
6. Test all changes properly after implementation.
7. Finish the implementation in one pass when possible.

First Step (mandatory):
Before writing code, inspect the existing app and identify:
- where this feature should live in the navigation
- what state management is already used
- what local storage / backend is already used
- what reusable widgets/components already exist
- what naming/style conventions the app follows

Then integrate the feature accordingly.

Feature to Build:
A `Goal Tracker` feature modeled after the `Books` sheet.

Core Experience:
- User can create a goal.
- Each goal has a target value.
- Each goal contains multiple tracked items.
- User can update item progress over time.
- The app automatically shows summary metrics and progress insights.

Examples of Goal Types:
- Read 10,000 minutes in 2026
- Read 50 books / 10,000 pages
- Exercise 5,000 minutes
- Save money toward a target
- Complete learning modules / lessons

Required Screens / UI:
1. Goal Summary / Dashboard
   - show goal title
   - show target, completed, remaining
   - show progress percentage
   - show average per day
   - show remaining days (if deadline exists)
   - show required per day to hit target
   - show estimated completion time based on current pace

2. Goal Details Screen
   - list all goal items
   - each item should show:
     - name
     - total amount
     - completed amount
     - progress %
     - optional remark/note
   - use simple progress visuals similar to the sheet

3. Add / Edit Goal Flow
   - create a goal title
   - set goal type or unit (minutes, pages, count, money, etc.)
   - set target amount
   - optionally set deadline

4. Add / Edit Item Flow
   - add item name
   - add total units
   - add current completed units
   - add note/remark

5. Update Progress Flow
   - allow quick progress update for each item
   - instantly refresh summary metrics after update

Metrics to Mirror from the Book Sheet:
Use the `Books` tab as inspiration for these calculated values:
- Completed
- Remain / Remaining
- Avg per Day
- Remaining Days
- Need per Day / Required per Day
- Days to Fill the Target / Estimated completion days
- Overall progress percentage

Suggested Data Model:
Goal
|- id
|- title
|- unitType
|- targetValue
|- completedValue
|- deadline
|- createdAt
`- items: List<GoalItem>

GoalItem
|- id
|- name
|- totalUnits
|- completedUnits
|- progressPercent
|- note
|- status
`- updatedAt

Integration Requirements:
- Add this feature into an appropriate existing section of the app (for example: Home, Productivity, Profile, Dashboard, or a Tools section).
- Reuse existing routing/navigation patterns.
- Reuse existing shared widgets if possible.
- Reuse current app colors/typography/components.
- If the app already has backend/data storage, use that.
- If not, use the app's current local persistence approach.
- Do not introduce unnecessary packages if current tools can already handle the feature.

Implementation Expectations:
- Read the current codebase first.
- Decide the smallest clean integration path.
- Add the feature with production-quality code.
- Keep files and naming consistent with the project.
- Avoid overengineering.
- Prefer a practical MVP that can be extended later.

Behavior Expectations:
- Progress updates should immediately affect summary numbers.
- Calculations should be correct and stable.
- Empty states should look clean.
- If there are no goals yet, show a friendly CTA to create one.
- If there are no items yet, show a helpful empty state in the goal details screen.

Testing Requirements:
After implementation, you must:
- run `flutter analyze`
- run relevant tests
- add or update tests for core calculations and main UI flows
- confirm there are no analyzer issues and tests pass

Output Requirements:
When done, provide:
1. a short summary of what was added
2. the list of files changed
3. verification results from `flutter analyze` and tests
4. any small follow-up suggestions only if necessary

Success Criteria:
- The new Goal Tracker is added inside the existing app.
- It reflects the style and logic of the `Books` sheet.
- Users can create goals, add items, update progress, and view meaningful metrics.
- The feature feels integrated, not bolted on.
- The code is clean, minimal, and tested.
